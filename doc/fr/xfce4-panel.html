À propos de xfce4-panel

 

Introduction

Le panneau Xfce 4 fait partie de l'environnement de bureau Xfce 4 et comprend des lanceurs de programmes, des menus, une horloge, un changeur de bureau, et plus encore. Beaucoup d'éléments de ce panneau peuvent être personnalisés via une interface graphique. Ce manuel devrait vous donner toutes les informations nécessaires pour adapter le panneau à vos préférences.

 

Démarrage

Normalement, vous lancerez le panneau comme élément de votre session Xfce avec le script startxfce4. La commande pour le lancer est xfce4-panel.

Quand vous lancez le panneau, vous voyez apparaître une barre en bas de votre écran, comme ceci :

Figure 1. Panneau Xfce par défaut

L'illustration montre la configuration par défaut du panneau avec un menu ouvert et la souris au-dessus d'un bouton lanceur, faisant apparaître un message d'aide. Le panneau Xfce utilise énormément les boutons gauche et droit de la souris. Cliquer sur un item avec le bouton gauche réalise l'action par défaut et avec le bouton droit, ouvre généralement un menu contextuel. Le clic droit peut être remplacé par un Maj + clic gauche. Notez que ce manuel se base sur une configuration pour droitier.

Il y a une poignée de déplacement à chaque extrémité du panneau. Vous pouvez les utiliser pour déplacer le panneau sur n'importe quel coté de l'écran. Déplacer le panneau sur plus de la moitié de l'écran le positionne sur le coté opposé. Cliquer avec le bouton droit sur l'une des poignées fait apparaître un menu popup... Allez-y, testez ;-)

Figure 2. Menu popup des poignées de déplacement

La zone située entre les poignées de déplacement contient les éléments du panneau. Les éléments disponibles sont décrits dans la section "Éléments du panneau". La plupart des éléments réalisent une action lors d'un clic. Par un clic droit sur un item, vous accédez à un menu popup permettant d'afficher ses propriétés, de supprimer l'item ou d'ajouter un nouvel item. Les items peuvent aussi être ajoutés avec un clic droit sur les poignées de déplacement (voir ci-dessus).

Figure 3. Èléments du menu popup

Quand vous voulez ajouter un nouvel élément, une fenêtre s'ouvre montrant tous les éléments disponibles. Certains éléments ne peuvent apparaître qu'en un seul exemplaire dans le panneau, le "system tray", par exemple, et sont donc affichés en grisé s'ils sont déjà présents.

Figure 4. Fenêtre d'ajout d'éléments

Les préférences globales du panneau peuvent être modifiées à partir du gestionnaire de paramètres Xfce 4. Celui-ci peut être lancé, soit à partir du bouton lanceur du panneau, soit par la commande xfce-settings-show. Voir "Préférences du panneau" pour une explication des options disponibles.

 

Items du panneau

Le panneau Xfce 4 peut charger aussi bien des plugins externes que internes. Dans cette partie, nous décrirons seulement les éléments qui sont installés en même temps que le panneau.

La fenêtre des propriétés d'un élément du panneau peut être ouverte avec un clic droit sur l'élément. Chaque type d'élément possède son propre dialogue de configuration. Chaque fenêtre de dialogue possède au minimum ces options :

  • Changer la position d'un élément dans le panneau.
  • Supprimer cet élément.
  • Fermer la fenêtre.

Lanceur

L'élément le plus courant est le lanceur. Il vous permet de démarrer un programme à partir du panneau. La fenêtre de dialogue propose les options suivantes :

Icône

Les icônes utilisées par le panneau respectent le thème d'icônes que vous avez choisi dans les préférences de l'interface utilisateur. Bien sûr, vous pouvez aussi sélectionner vos propres icônes en parcourant les répertoires grâce au bouton à coté de la zone de saisie, ou en déposant une image à partir du gestionnaire de fichier sur la zone de prévisualisation.

Commande

Vous pouvez entrer la commande à lancer ou parcourir le système de fichiers. Il existe une option pour lancer un programme dans un terminal et pour utiliser la notification de démarrage. Cette dernière option signifie que le gestionnaire de fenêtres peut montrer un sablier pendant le chargement de l'application, à condition que celle-ci supporte cette fonctionnalité. Les programmes basés sur Gtk 2.2 le permettent.

Tooltip

The panel item will show this tooltip if you hold the mouse over it.

Menu

Un lanceur peut contenir un menu. Quand cette option est choisie, un bouton avec une petite flêche apparaît à coté de l'élément. Reportez-vous à menus du panneau pour plus d'information. La position du bouton peut être modifiée dans les préférences générales.

Figure 5. Éléments du lanceur

Horloge Xfce

Le style de l'horloge Xfce peut être modifié par certaines options. Il existe trois types d'horloge possibles : analogique, numérique et LED. Une option permet d'afficher ou non les secondes, et pour les modèles numérique et LED vous pouvez choisir un affichage sur 24 ou 12 heures (avec possibilité d'ajouter un mode AM/PM). L'horloge affiche en outre la date en bulle d'aide. Cliquer sur l'horloge n'apporte rien.

Figure 6. Élément horloge

Contrôle de courrier

Le moniteur de courrier surveille l'arrivée de courrier dans la boîte aux lettres. Celle-ci peut être au format mbox ou maildir.

L'icône de courrier s'accorde avec votre thème d'icônes. Vous pouvez choisir l'emplacement de votre boîte, la commande à lancer lorsque vous cliquez sur le bouton et l'intervalle entre les vérifications. La commande a les mêmes options que celles des éléments lanceurs, voir au-dessus).

Figure 7. Élément contrôle de courrier

Pour une utilisation avancée, vous pouvez aussi indiquer une boîte aux lettres distante en utilisant le format suivant : pop3://user:password@server:port. Si omettez le port, ce sera 110 par défaut

Notez que cette fonctionnalité est expérimentale. Avant de l'utiliser, vérifiez que votre connection réseau est bien établie, sinon une erreur du moniteur de courrier peut bloquer le panneau.

Changeur de bureaux et changeur de bureaux graphique

Le panneau fournit deux éléments pour interagir avec les bureaux virtuels. Un changeur d'espace de travail de type CDE qui montre des boutons avec le nom des bureaux, et un changeur graphique qui affiche une vue miniature du bureau et vous permet même de déplacer une fenêtre vers un autre espace par un glisser-déposer. Le changeur graphique permet aussi de passer d'un espace à l'autre avec la molette de la souris.

Dans le cas du changeur de bureaux graphique, vous pouvez indiquer le nombre de rangs à utiliser, ou de colonnes quand le panneau est en mode vertical.

Figure 8. Changeur de bureaux graphique

Boutons système

Un élément "Boutons sytème" possède un ou deux boutons qui peuvent réaliser des actions système spécifiques. Le bouton s'accorde avec votre thème d'icône. Il existe quatre boutons possible :

Verrouiller

Lance le script xflock4 qui tente de verrouiller l'écran soit avec xscreensaver, soit, en cas d'échec, avec xlock.

Quitter

Ceci ouvre une fenêtre de déconnexion pour terminer votre session Xfce.

Info

Le bouton info ouvre la fenêtre À propos de Xfce 4.

Paramètres

Le bouton Paramètres ouvre le gestionnaire de paramètres de Xfce, à partir duquel, vous accèdez aux réglages de tous les composants de Xfce, ainsi qu'à certains réglages système, tels le comportement de la souris et les paramètres de l'interface utilisateur.

Figure 9. Élément bouton système

Séparateur

Cet élément ajoute simplement une ligne de séparation sur le panneau. Il n'a pas d'autre option que sa position.

Notez qu'il peut être difficile d'ouvrir son menu popup car vous disposez de trés peu de surface sur laquelle cliquer.

Figure 10. Élément sans option

 

Menus du panneau

Les éléments lanceurs du panneau peuvent être associés avec un menu, lequel peut être ouvert en cliquant sur le bouton fléché à coté. La position de ce bouton est configurable dans les préférences générales. Vous pouvez aussi ouvrir le menu en gardant enfoncé le bouton lanceur. Le menu s'ouvre aprés un court délai.

Utilisez le choix 'Ajouter un lanceur' en première ligne du menu du lanceur, ou déposez un exécutable à partir du gestionnaire de fichier sur cette ligne, pour ajouter un nouveau choix au menu. Pour supprimer un élément du menu, utilisez le bouton 'Supprimer' dans la fenêtre des propriétés de l'élément.

Une possibilité particulière des menus du panneau est qu'ils peuvent être détachés du panneau et rester dans leur propre fenêtre. Pour détacher un menu, cliquer sur la petite ligne en bas du menu (voir la copie d'écran du panneau par défaut).

Le dialogue des propriétés d'un élément du menu s'ouvre de suite quand vous faites un clic droit sur celui-ci. Cette fenêtre de dialogue est trés semblable à celle de l'élément lanceur.

Figure 11. Élement de menu

 

Préférences du panneau

La fenêtre des propriétés du panneau Xfce 4 peut être ouverte à partir du gestionnaire de paramètres de Xfce 4 ou par un clic droit sur une des poignées de déplacement du panneau.

Figure 12. Paramètres du panneau

Apparence

Taille

Quatre tailles sont disponibles : petit, moyen, grand, énorme. La taille par défaut est moyen.

Orientation

Le panneau Xfce 4 peut être placé aussi bien en position verticale que dans la position commune horizontale.

Position du bouton d'ouverture

Cette option modifie la position des boutons qui ouvrent un menu du panneau. Ils peuvent être au-dessus, au-dessous, à gauche ou à droite de l'élément associé.

Comportement

Masquer automatiquement

Si vous choisissez de le masquer automatiquement, le panneau se transforme en une fine barre. Passez la souris dessus le démasque.

 

Avancé

Variables de fichiers et d'environnement

Xfce utilise maintenant la spécification de répertoire définie par Freedesktop.org pour localiser ses fichiers de données et de configuration. Cela signifie que l'emplacement des fichiers sera indiqué comme un chemin relatif aux répertoires décrits dans la spécification.

${XDG_CONFIG_HOME}

Le premier répertoire de base dans lequel chercher les fichiers de configuration. Par défaut, c'est ~/.config/.

${XDG_CONFIG_DIRS}

Une liste de répertoires système qui contiennent les données de configuration. Par défaut le panneau va chercher dans ${sysconfdir}/xdg/ et /etc/xdg/. La valeur de ${sysconfdir} dépend de la façon dont le programme a été compilé et sera souvent /etc/ pour les paquetages binaires.

<basedir>/xfce4/panel/contents.xml

Ceci est l'emplacement du fichier de configuration xml qui décrit le contenu du panneau, relativement au répertoire de base mentionné ci-dessus.

<basedir>/xfce4/mcs_settings/panel.xml

Ceci est l'emplacement relatif des paramètres du panneau. Ce fichier est géré par le gestionnaire de paramètres, plutôt que par le panneau lui-même.

<basedir>/xfce4/kiosk/kioskrc

Fichier de configuration du mode kiosk. Voir la prochaine section pour une explication.

<basedir>/xfce4/transparency

Configuration de la transparence. Voir une explication complémentaire dans cette section plus loin.

Aucun des fichiers de configuration, à l'exception de kioskrc et transparence, ne sont conçus pour être édité manuellement. En fait, si vous les éditez pendant que le gestionnaire de session ou le gestionnaire de paramètres fonctionnent, les modifications seront écrasées.

Mode kiosk

Le panneau offre maintenant un support pour le mode kiosk, qui aide à empêcher les utilisateurs de modifier la configuration de leur panneau. Pour l'utiliser, vous devez éditer ou créer le fichier ${sysconfdir}/xdg/xfce4/kiosk/kioskrc.

La meilleure manière d'expliquer le format de ce fichier est d'utiliser un exemple. La section xfce4-panel de votrekioskrc peut se présenter comme ceci : 

[xfce4-panel]
CustomizePanel=%powerusers,foo

Ceci autorise une personnalisation du panneau uniquement aux utilisateurs du groupe powerusers et à l'utilisateur foo.

Transparence

Notez, s'il vous plaît, que la transparence ne fonctionne qu'avec un serveur X qui comprend l'extension Composite, tel XOrg >= 6.8.0 et si l'extension Composite a été activée dans le fichier de configuration. Et si le gestionnaire de fenêtres, aussi, comprend cette extension (par exemple xfwm4 doit être compilé avec l'option --enable-compositor).

Par défaut, le panneau est transparent à moins que l'option masquer automatiquement soit activée. Lorsque la souris survole le panneau, la transparence est suspendue momentanèment.

Vous pouvez créer le fichier <basedir>/xfce4/transparency si vous souhaitez modifier la valeur de transparence du panneau (ou de la boîte à icônes). Créez ce fichier et ajouter une ligne de type panel=<value>, où value est un nombre entre 0 et 100. Avec 0, la transparence est désactivée.

Par exemple, un fichier de transparence peut ressembler à ceci : 

panel=25
iconbox=0
